# 🌐 Industrial Talk: Telekom Malaysia (TM) Strategic Tech Talk

## 📌 Talk Summary
* **Organization:** Telekom Malaysia (TM) Berhad
* **Objective:** To analyze national-scale telecommunication infrastructures, high-throughput network streaming datasets, and massive distributed data lake architectures.
* **Key Focus:** High-Performance Data Processing (HPDP), real-time network traffic telemetry, fraud detection pipelines, and elastic database management strategies.

This strategic technical engagement with Telekom Malaysia provided a rare look into how a national telecom giant ingests, orchestrates, and processes petabytes of unstructured text logs, call detail records (CDRs), and cellular telemetry daily. The session highlighted the real-world application of distributed computing clusters to achieve minimal latency while executing concurrent analytical models for proactive network maintenance and automated fraud risk mitigation.

## 🧠 Reflection

### 1. What I Learnt
* The Telekom Malaysia industrial presentation redefined my understanding of "Big Data." While typical classroom assignments work with static relational database files sizing a few megabytes, TM operates at a scale where data streams continuously at gigabytes per second from millions of connected endpoints and IoT nodes nationwide.

* I learned that traditional processing configurations fail instantly under this type of workload. At this scale, it is critical to implement **High-Performance Data Processing (HPDP)** architectures. Data teams must leverage distributed parallel file structures and partitioned stream layers to divide the massive computing load among independent worker nodes. This methodology ensures data can be transformed and checked for errors in real-time before hitting internal analytic pools.
